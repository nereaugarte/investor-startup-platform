## Technology Decisions

### Why Serverless?
- **Cost-effective:** Pay only for what you use
- **Scalable:** Auto-scales from 0 to thousands of users
- **No server management:** Focus on code, not infrastructure
- **High availability:** Built-in redundancy

### Why Step Functions?
- **Orchestration:** Coordinates multiple Lambda functions
- **Visibility:** Easy to track workflow execution
- **Error handling:** Built-in retry logic
- **State management:** Maintains workflow state

### Why DynamoDB?
- **Performance:** Single-digit millisecond latency
- **Scalability:** Scales horizontally automatically
- **Serverless:** No capacity planning needed
- **Cost-effective:** On-demand pricing

### Why SES?
- **Reliable:** 99.9% uptime SLA
- **Cost-effective:** $0.10 per 1,000 emails
- **HTML support:** Rich email formatting
- **Bounce handling:** Automatic bounce management
